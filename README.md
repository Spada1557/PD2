# Столовая+

`Столовая+` — статическое веб-приложение для предварительного заказа готовых блюд в студенческой столовой. Посетитель выбирает блюда из доступного ассортимента, указывает слот выдачи и получает номер заказа. Сотрудник столовой видит заказы, меняет их статусы и управляет наличием блюд в меню.

## Возможности

- регистрация и вход посетителя;
- просмотр меню по категориям;
- добавление блюд в корзину;
- выбор времени выдачи из доступных слотов;
- оформление заказа с комментарием;
- просмотр истории и статуса заказов;
- рабочая панель сотрудника для комплектования заказов;
- включение и отключение блюд в меню.

## Запуск

Откройте файл `app/index.html` в современном браузере.

Приложение не требует сервера, сборки и установки зависимостей. Данные учебного прототипа сохраняются в `localStorage` браузера.

## Демо-доступ сотрудника

```text
логин: admin
пароль: admin123
```

Обычный посетитель может создать аккаунт через форму регистрации.

## Структура приложения

```text
app/index.html              основная страница приложения
app/assets/app.js           логика пользователей, меню, корзины и заказов
app/assets/styles.css       стили интерфейса
app/assets/images/          логотип и изображения блюд
app/vendor/vue.global.prod.js локальная сборка Vue
```

## Для сопровождающих программистов

- Проект написан на HTML, CSS и JavaScript с Vue без этапа сборки.
- Начальные пользователи, блюда, статусы и слоты выдачи находятся в `app/assets/app.js`.
- Ключи локального хранилища имеют префикс `pd2.cafeteria`.
- Для сброса демо-данных очистите `localStorage` сайта в браузере.
- При переносе на серверную архитектуру локальное хранилище можно заменить API, сохранив текущую предметную модель: пользователи, блюда, заказы, позиции заказа и статусы.
